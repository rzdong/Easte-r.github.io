EventSource 服务器发送事件

定义​

一个可以接收服务器发送事件的Web接口，只能由服务器向Client推送数据

该实例打开与HTTP服务器的持久连接,直到调用EventSource.close()后关闭

正常来说，HTTP协议无法做到服务器主动推送信息。但是有一种变通方法，就是服务器向客户端声明接下来要发的信息是流信息，这时客户端不会关闭连接，会一直等待服务器发送新的数据流。本质就是以流信息的方式，完成一次用时很长的下载任务。

与Server-sent events（SSE）相关

SSE与WebSocket区别​

SSEWebSocket
SSE使用HTTP协议WebSocket是一个独立的协议
SSE轻量级，使用简单WebScoket协议相对复杂
SSE默认支持断线重连WebSokcet需自己实现
SSE一般用来传输文本，二进制数据需要编码后传送WebSocket默认支持传送二进制数据
SSE支持自定义发送的消息类型-

浏览器API EventSource​

new EventSource(url)
new EventSource(url, {
  withCredentials: false // default
})

属性​

属性描述
readyState一个只读属性，表示当前连接状态
0 - connecting
1 - open
2 - closed
url请求url
withCredentials跨域时是否携带cookie

方法​

方法描述
closeEventSource示例调用close方法后会关闭连接，并且将readyState状态设置为2

事件​

事件描述(事件无法取消、冒泡)
error服务器连接无法打开，将会触发错误事件
message服务器返回数据时触发
open与服务器连接时触发

基本用法​

1. 客户端初始化EventSource​

let source;
if ('EventSource' in window) {
  source = new EventSource('http://127.0.0.1', {
    withCredentials: true, // 跨域时也可以发送cookie，默认false
  });
} else {
  document.write('当前设备不支持EventSource');
}

2. 客户端监听事件​

let data = '';
source.addEventListener('open', function (event) {
  console.log('连接已打开')
}, false);

source.addEventListener('message', function (event) {
  let d = event.data; // 字符串
  data += d;
  console.log(data);
}, false);

source.addEventListener('error', function (event) {
  // 连接发生错误，可能是网络问题，也可能是服务器不可用
}, false);

source.close(); // 在一定时间内没发送完数据，客户端可以主动提前关闭连接

3. 客户端自定义事件​

默认情况下，服务器发来的数据，会触发EventSource实例的 message 事件，但服务器也可以自定义事件，此时发送回的数据不会触发 message 事件

source.addEventListener('ping', function (event) {
  console.log(event.data); // event.data 是字符串
}, false);

4. 服务器配置响应头​

// 服务器需要响应如下响应头
Content-Type: text/event-stream // 字符串的事件流类型
Cache-Control: no-cache // 不缓存

5. 服务器发送消息​

信息

每行消息由一行或者多行文字组成，列出该消息的字段，每个字段由字段名表示，后面是冒号，然后是该字段的文本值 规范中有以下字段

  • event 一个用于标识事件类型的字符串，制定了这个字符串。EventSource实例必须监听此事件，而不是 message; 如果一个消息没有指定事件名，那么就默认为 message
  • data 消息的内容，当EventSource接收到多个以 data: 开头的连续行时，会将他们连接起来，在他们之间插入换行符，末尾的换行符会被删除
  • id 事件ID，唯一标识此事件
  • retry 整型，毫秒时间戳。表示重新连接的时间，如果与服务器的连接丢失，浏览器将等待指定的时间，然后尝试重新连接
提示

如果一行文本中不包含冒号，则整行文本会被解析成为字段名，其字段值为空。如下 如果一个消息以冒号开头时是注释行，会被忽略，可以用来防止连接超时，服务器定期发送一条消息注释，能保持连接不断

this is a test stream

// or

: this is a test stream

这条消息会被理解为一个字段名，不合法的字段名都会被忽略 以冒号开头的行为注释行，会被忽略

data字段​

数据内容用data字段表示

data: message\n\n

id字段​

数据的唯一标识用id表示，如果断开连接，浏览器会发一个HTTP头，里面携带了一个特殊的 Last-Event-ID 信息，服务器将根据此信息重建连接并同步信息

id: fd4321hf\n
data: message\n\n

event字段​

event字段默认缺省是 message 客户端可以用 addEventListener 监听此字段定义的事件

event: ping\n
id: 43ghfdu3\n
data: message\n\n

retry字段​

服务器可以用retry字段，告诉浏览器断连后重新发起连接的时间间隔（毫秒） 连接出错或者retry时间到期，浏览器都会发起重连

源代码​

信息

源代码参考 Server-Sent Events 教程

node 服务端​
var http = require("http");

http.createServer(function (req, res) {
  var fileName = "." + req.url;
  console.log(req.rawHeaders) // 当收到请求时，打印Header
  if (fileName === "./stream") {
    res.writeHead(200, {
      "Content-Type":"text/event-stream",
      "Cache-Control":"no-cache",
      "Connection":"keep-alive",
      "Access-Control-Allow-Origin": '*',
    });
    res.write("retry: 10000\n");
    res.write("event: connectinit\n");
    res.write(`id: ${Math.random().toString().slice(2)}\n`);
    res.write("data: " + (new Date()) + "\n\n"); // 发送一条初始化消息

    res.write("data: " + (new Date()) + "\n\n"); // 单独发送第二条消息
    res.write("data: " + (new Date()) + "\n\n"); // 单独发送第三条消息

    interval = setInterval(function () {
      res.write(`id: ${Math.random().toString().slice(2)}\n`); // 每两秒发一条消息，带有不同的id
      res.write("data: " + (new Date()) + "\n\n");
    }, 2000);

    req.connection.addListener("close", function () {
      console.log('connect-close')
      clearInterval(interval);
    }, false);
  }
}).listen(8844, "127.0.0.1");
客户端​
<!DOCTYPE html>
<html>
<head>
  <meta charset="utf-8">
  <meta name="viewport" content="width=device-width">
  <title>JS Bin</title>
</head>
<body>
<div id="example"></div>
<script>
  var source = new EventSource('http://127.0.0.1:8844/stream');
  var div = document.getElementById('example');
  
  source.onopen = function (event) {
    div.innerHTML += '<p>Connection open ...</p>';
  };
  
  source.onerror = function (event) {
    div.innerHTML += '<p>Connection close.</p>';
  };
  
  source.addEventListener('connectinit', function (event) { // 第一个请求是 'connectinit' 事件
    console.log('init:' + event.lastEventId) // 来自服务端的第一个请求
    div.innerHTML += ('<p>Start time: ' + event.data + '</p>');
  }, false);
  
  source.onmessage = function (event) { // 默认监听 'message' 事件
    console.log(event.lastEventId) // 打印来自服务端的eventId，在客户端尝试重新连接时，会发起一个新的HTTP请求，其中的header带有lasteventid
    div.innerHTML += ('<p>Ping: ' + event.data + '</p>');
  };
  
</script>
</body>
</html>

6. 效果如下​

页面打开后

  1. 首先触发了 open 事件
  2. 接着触发了自定义的 connectinit 事件
  3. 接着触发了默认的 message 事件 console面板里有收到事件时的EventId

服务器关闭后

  1. 触发了 error 事件
  2. 等待10秒后，重新发起Http请求尝试重连，此时携带了一个 Last-Event-ID, 即是之前收到的最后一个事件的EventId

服务器重启后

  1. 等到retry设置的重试时间到期时，继续发起同样的EventSource请求
  2. 服务器收到对应的请求，并且请求的Header里有一个特殊的 Last-Event-ID。（服务器可以基于这个ID做断线重连操作）
