getComputedStyle 获取样式表

window.getComputedStyle​

getComputedStyle方法返回一个只读对象CSSStyleDeclaration对象，该对象包含给定的element的所有CSS属性，这些属性是当前element所应用的最终属性。

各个CSS属性值可以通过对象提供的API，或者使用CSS属性来访问，比如

const style = window.getComputedStyle(document.body);
// 通过css属性直接访问属性值
console.log(style.color) // 'rgb(255, 255, 255)'

// 通过getPropertyValue 传入指定的css属性获取属性值
console.log(style.getPropertyValue('color')) // 'rgb(255, 255, 255)'
提示

有些三方库，比如jquery。调用getComputedStyle方法是从document.defaultView.getComputedStyle获取，实际上和window.getComputedStyle完全相同

console.log(document.defaultView === window) // 输出true

window.getComputedStyle方法的参数​

方法描述
element需要被计算属性的DOM节点
pseudoElt指定要匹配的伪元素，使用这个参数可以指定element的伪类（可选项，默认null, ）

调用方式如下

window.getComputedStyle(document.body);

window.getComputedStyle(document.body, ':after');
提示

建议总是给getComputedStyle方法传入第二个参数, 不需要伪类就传null
因为Firefox3-3.6版本中，第二个参数是必填的CanIUse getComputedStyle

getComputedStyle(element) 与 element.style​

他们都会得到同样的CSSStyleDeclaration对象, 但有一些区别

  1. getComputedStyle 获取的属性是只读的，如果通过属性手动设置会得到报错，如下图


    直接通过 `element.style` 设置css属性可以成功生效，页面文字也变成了红色

  2. [element].style 属性用于给element元素设置样式

  3. [element].style 只能获取到设置在该元素的style属性上的css样式。而getComputedStyle可以获得一个元素最终被应用的所有样式

兼容性​

备注

兼容性参考张鑫旭的文章获取元素CSS值之getComputedStyle方法熟悉

element元素，除了拥有style属性，还可能会拥有currentStyle属性

style 属性返回的是当前元素通过style设置的属性，如果是css样式文件设置的就无法获取到

currentStyle 属性返回当前元素计算后的最终css属性，其返回的结果和getComputedStyle是比较相似的 （该属性ie浏览器支持）

所以在获取一个元素的完整Style计算属性时可以这样兼容

// 如果元素不存在currentStyle属性，则使用getComputedstyle获取
const styles = element.currentStyle || window.getComputedstyle(element, null)

style.getPropertyValue 方法​

getPropertyValue​

方法属于 CSSStyleDeclaration 实例，通过此方法可以获取到实例中的某个CSS属性，比如

const style = window.getComputedStyle(document.body);
console.log(style.getPropertyValue('color')) // 'rgb(255, 255, 255)'

获取元素的CSS变量​

除此以外，getPropertyValue方法还可以获取元素上被设置的css变量，比如：

<style>
  body {
    --color: red;
    color: var(--color);
  }
</style>

<body>
  content
</body>
<script>
  const styles = window.getComputedStyle(document.body);
  console.log(styles.getPropertyValue('--color')) // 最终输出结果 'red'
</script>

设置元素属性​

由于 getComputedStyle 方法获取到的style只能读取属性，所以需要设置style属性时，要使用 element.style, 比如：

// 设置body的color为'red'。第一种
document.body.style.color = 'red';

// 设置body的color为 'red'。第二种
document.body.style.setProperty('color', 'red')

// 设置body的css变量'--color'为'red'
document.body.style.setProperty('--color', 'red')

// 特别注意，不能直接通过style直接设置css变量，下方的设置无效
document.body.style['--color'] = 'red' // 无效

getPropertyValue兼容性​

getPropertyValue方法也有兼容性问题，在IE-8及以下不支持，作为替代，可以使用 style.getAttribute 方法代替

应用​

1. 获取伪元素的属性值​

<style>
  h3::after {
    content: " rocks!";
  }
</style>

<h3>Generated content</h3>

<script>
  const h3 = document.querySelector("h3");
  const result = getComputedStyle(h3, ":after").content;

  console.log("the generated content is: ", result); // returns ' rocks!'
</script>

2. html2canvas​

html2canvas是一个可以把html元素转化为图片的库，其内部就是使用了getComputedStyle获取dom元素的属性开始绘制canvas

3. 配合css媒体查询any-hover 检测当前用户设备是否支持 鼠标:hover​

/* 
  any-hover 是一个媒体查询特性，值为none，或者hover
  可以检查当前设备是否支持 否有任意可用的输入机制可以在元素上hover
*/

下方媒体查询的意思是：如果当前设备不支持none，那么body的::before伪类就会被设置一个content: hoverNone
@media (any-hover: none) {
    body::before {
        content: 'hoverNone';
        display: none;
    }
}
// 此时可以在脚本中调用方法检查body的伪类的content属性，如果为

var strContent = getComputedStyle(document.body, '::before').content;
if (strContent === 'hoverNone') {
  console.log('当前设备不支持伪类')
}

4. 配合css媒体查询max-width​

提示

此案例来自：伪类+js实现CSS3 media queries跨界准确判断

假设有一个自适应的网页，需要对1024px为分割线做自适应：宽度小于1024px的，看作移动端；宽度大于1024px的看作PC端，除了CSS的媒体查询自动做响应式，有些逻辑也需要在js代码里做判断

一、常规做法如下

/* css写一套样式 */
@media screen and (max-width: 1024px) {
    img {
        width: 128px;
        height: 128px;
    }
}
// js里对1024px做同样的判断
window.addEventListener("resize", function() {
    // window.innerWidth IE9+浏览器支持
    if (window.innerWidth <= 1024) {
        // 普屏处理
    } else {
        //宽屏处理
    }
}, false);

如果上图中1024的值变动，比如重新考虑以768px为分界线。就需要改动css + js，可以改为如下做法

二、获取CSS当前媒体查询的状态值


/* 默认是移动端 */
body::after {
  display: none;
  content: 'mobile';
}

/* 如果符合规则，则设置content为pc端 */
@media screen and (min-width: 1024px) {
    body::after {
        display: none;
        content: 'pc';
    }
}
let plantform = 'mobile'; // 当窗口变化的时候，检查body的伪类的content内容
window.addEventListener("resize", function() {
  plantform = window.getComputedStyle(document.body, ':after').getPropertyValue('content')
}, false);

扩展​

CSSStyleDeclaration 对象​

有三种API可以获取到CSSStyleDeclaration对象

  1. 通过 HTMLElement.style 获取，获取的对象里只有当前元素的内联style样式，形如 <div style="...">, 该属性可读可写

  2. 通过 Window.getComputedStyle 获取，获取的对象是只读的。无法更改。

  3. 通过 CSSStyleSheet API获取， 可以通过此路径访问到 document.styleSheets[0].cssRules[0].style

CSSStyleSheet 对象​

CSSstyleSheet 对象标识单个CSS样式表，并且允许更改其中的CSS样式

此对象可以通过 document.styleSheets 获取到，如下图

styleSheets 返回一个 StyleSheetList 数组，其中每一个数组元素都是一个 CSSStyleSheet，即一个由link或者style引入的样式表

CSSStyleSheet 表示一个样式表的所有信息，其中的 cssRules 属性表示这个样式表里的一系列选择器+CSS属性

这些数组元素代表着DOM上的三个css样式表。上面截图中有三个属性，表示当前网页引入了三个样式表:

<!-- 数组的顺序代表着在样式引入的顺序 -->
<style media="print">.article-actions-container,.document-toc-container,.language-menu,.main-menu-toggle,.mdn-cta-container,.on-github,.page-footer,.place,.sidebar,.top-banner,.top-navigation-main,ul.prev-next{display:none!important}.main-page-content,.main-page-content pre{padding:2px}.main-page-content pre{border-left-width:2px}</style>
<link href="/static/css/main.b9154aa3.css" rel="stylesheet">
<style>.jj-flash-note__popper[data-v-44225974]{position:absolute;border:none;outline:0;text-align:center;width:28px;height:28px;}</style>

他们都属于样式表，可以通过 CSSStyleSheet.ownerNode 获取对应的样式的所属的节点类型

  • object HTMLStyleElement 由style标签引入的样式
  • object HTMLLinkElement 由link标签引入的样式

CSSStyleSheet 属性​

  1. cssRules返回一个实时的CSSRuleList，其中，每个数组元素表示该样式表中的一个CSS选择器+CSS属性

下图中cssRules包含了三个选择器。

  • cssText显示了完整的 CSS选择器+CSS属性 的内容

  • selectorText 显示了当前规则的CSS选择器

  • style 即为 CSSStyleDeclaration对象 可读写 通过以下方式直接访问到style，并且设置对应的属性，可以即时的改变当前网页（当前网页的fontSize被设置为指定值）

ownerNode 拥有此样式表的dom节点，可能是<link>，也可能是<style>，<?xml-stylesheet ?>

ownerRule 表示当前样式表是否被其他样式表使用 @import 引入

rules 即将被废弃，功能与cssRules完全相同，用cssRules代替即可

href 如果是<link>引入的样式，href表示<link>的href
